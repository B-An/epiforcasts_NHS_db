# Agent Instructions

This folder uses concise execution rules to improve coding quality and reduce rework.

## Karpathy Execution Rules (Lightweight)

Use these rules for non-trivial coding tasks:

1. Think before coding: state assumptions, ask when ambiguous, surface tradeoffs.
2. Simplicity first: solve only the requested problem; avoid speculative abstractions.
3. Surgical changes: touch only lines required for the task; no unrelated cleanup.
4. Goal-driven execution: define verifiable success criteria before editing.
5. Verify loop: reproduce issue, implement minimal fix, run targeted checks/tests.
6. Trivial-task exception: for obvious one-line fixes, proceed directly.
